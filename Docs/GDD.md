## 1. Game Concept

### 1.1 High concept
지팡이에 스펠을 조합해 커스터마이징하고, 턴제로 발사체를 발사하여 벽돌을 파괴하며, 노드맵 기반의 스테이지를 진행하는 로그라이크.

### 1.2 References

| Reference        | Borrowed element                               | Note                          |
|------------------|------------------------------------------------|-------------------------------|
| Noita / MagicCraft | 지팡이 스펠 커스터마이징 시스템                | 빌드 시스템의 핵심            |
| Arkanoid         | 발사체 반사각 물리 + 벽돌 파괴                 | 인게임 코어 메카닉            |
| Slay the Spire   | 노드맵 스테이지 진행 + 빌드업 구조             | 메타 진행 및 리플레이 구조    |

### 1.3 Target
- **Platform**: PC (Steam)
- **Target playtime**: 30–60분 / 1런
- **Audience**: 로그라이크/덱빌더 장르 팬, 전략적 빌드 조합을 즐기는 플레이어

## 2. Core Loop

플레이어가 반복하는 핵심 행동 사이클:

| Step | Action       | Description                                                              |
|------|--------------|--------------------------------------------------------------------------|
| 1    | 발사 준비    | 각도 / 위치 조준                                                         |
| 2    | 시전         | 지팡이 스펠 시퀀스 순차 발동, 발사체가 벽돌과 충돌 / 반사 / 관통        |
| 3    | 결과 처리    | 벽돌 파괴, 드롭 획득, 점수 계산                                         |
| 4    | 적 턴        | 모든 벽돌 1줄 하강 + 상단에서 새 줄 생성                                |
| 5    | 피해 판정    | 최하단(위험 라인) 도달 벽돌 → 플레이어에게 데미지                       |
| 6    | 다음 턴 준비 | 마나 회복, 쿨다운 해제 등                                               |

### 2.1 Turn definition
한 턴 = **지팡이 1회 시전**. 지팡이 안의 스펠 시퀀스가 순차 발동되고, 모든 발사체가 소멸하면 턴 종료.

### 2.2 Victory / Defeat
- **승리**: 웨이브 기반. N웨이브 동안 벽돌 생성 후 생성 중단, 남은 벽돌 전부 처리 시 승리.
- **패배**: 플레이어 HP가 0이 되면 런 종료. 벽돌이 위험 라인 도달 시 도달 벽돌 수 × 기본 데미지.

## 3. Play Field

### 3.1 Field size
기본 필드: **8칸(가로) × 15칸(세로)**. 보스전이나 기믹에 따라 변동 가능 (6×15, 10×15 등).

### 3.2 Field layout

| Area        | Row    | Description                                  |
|-------------|--------|----------------------------------------------|
| 생성 영역   | 1      | 매 턴 새 벽돌 줄 진입                        |
| 전투 공간   | 2–13   | 벽돌 하강 및 발사체 운동 영역 (12줄)         |
| 위험 라인   | 14     | 이 아래 도달 시 플레이어 피해                |
| 플레이어    | 15     | 조준 및 발사 위치                            |

### 3.3 Physics
- 기본 반사: 입사각 = 반사각 (벽, 벽돌 측면)
- 좌우 벽 및 상단 벽에서 반사
- 벽돌 충돌 시: 데미지 적용 + 반사. HP 0 시 파괴, 발사체 진행 지속
- 발사체는 바운스 횟수 초과 또는 하단 이탈 시 소멸
- 스펠에 따라 물리 특성 변형 (관통, 유도, 직선 등)

자세한 충돌 / 반사 / 시전 평가 모델은 `Systems/Combat.md`.

## 4. Brick System

### 4.1 Properties
- **HP**: 1–3 히트
- **속성**: 무, 화염, 냉기, 전기 (실드 벽돌과의 상성용)
- **드롭**: 재화, 스펠, 회복

### 4.2 Special brick types

| Type        | Effect                                 | Strategic role                |
|-------------|----------------------------------------|-------------------------------|
| 실드 벽돌   | 특정 속성에만 피해                     | 빌드 다양성 강제              |
| 폭발 벽돌   | 파괴 시 주변 벽돌 함께 파괴            | 체인 반응 유도                |
| 스폰 벽돌   | 파괴하지 않으면 매 턴 주변에 벽돌 추가 생성 | 우선 처리 타겟           |
| 반사 벽돌   | 발사체를 튕겨냄                        | 각도 계산의 전략적 요소       |
| 보스 벽돌   | 고HP + 특수 패턴                       | 엘리트 / 보스 스테이지 전용   |

### 4.3 Line generation
줄 생성 규칙이 곧 난이도 커브:

- **초반 액트**: 빈 칸 많음, HP 1 위주, 특수 벽돌 비율 낮음
- **후반 액트**: 밀도 증가, 고HP 비율 증가, 특수 벽돌 비율 증가
- **보스전**: 보스가 줄 생성 패턴 자체를 결정 (보스의 공격 패턴 역할)

세부 규칙은 `Systems/Brick.md`.

## 5. Character System

### 5.1 Design philosophy
캐릭터는 등장 스펠 풀을 결정하고, 지팡이는 공용 장비로 독립되어 있습니다.

- **캐릭터**: 고유 패시브 1개 + 전용 스펠 풀
- **지팡이**: 슬롯 수 / 시전 방식 (모든 캐릭터 공용)
- 스펠 풀 = 공용 스펠(~25종) + 캐릭터 전용 스펠(8~10종)
- 해금 시스템으로 전용 풀이 단계적으로 확장

### 5.2 Gunslinger (base character)
- **Passive — 정밀 사격**: 발사체가 바운스 없이 첫 충돌 시 데미지 +1
- **Concept**: 속성 / 상태이상 없이 순수 물리 데미지와 궤적 조작으로 승부. 조준 실력과 반사각 이해도가 곧 성능.

## 6. Weapon System

> v0.1의 무기는 모두 **지팡이 형태**(Gunslinger 캐릭터). 향후 다른 캐릭터의 무기 카테고리(총포, 지팡이가 아닌 도구 등) 도입을 위해 코드/스펙 식별자는 일반화된 `Weapon` 을 쓴다.

### 6.1 Weapon structure
지팡이는 캐릭터와 독립된 공용 장비입니다. 슬롯 수와 시전 방식으로 차별화되며, 런 중 상점 / 보상에서 교체 가능합니다.

### 6.2 Spell slot layout

```
[Trigger] → [Projectile] → [Modifier] → [Modifier] → [Effect]
```

순서가 중요합니다. **Modifier는 바로 앞의 Projectile**(또는 앞선 Modifier의 결과)을 변형합니다. 평가 모델 상세는 `Systems/Combat.md` 와 `Systems/Spell.md`.

### 6.3 Initial weapon set (3 types)

| Name             | Slots | Cast policy                  | Trait                                                              |
|------------------|-------|------------------------------|--------------------------------------------------------------------|
| 수습의 지팡이    | 4     | 턴당 1회                     | 제한 없음. 기본 지팡이                                             |
| 연발 지팡이      | 3     | 턴당 2회                     | 슬롯 적지만 같은 시퀀스 2번 시전. 각각 다른 발사각 설정 가능       |
| 대마도의 지팡이  | 5     | 턴당 1회, 시전 후 1턴 쿨다운 | 긴 스펠 체인으로 강력한 한 방. 격턴 시전이라 빈 턴 리스크         |

## 8. Relic System

### 8.1 Overview
유물은 Slay the Spire의 렐릭과 동일한 역할의 패시브 효과입니다. 스펠 조합이 전술이라면, 유물은 전략으로서 런의 빌드 방향을 결정해줍니다.

- 획득 경로: 엘리트 클리어(2중 1), 보스 클리어(3중 1), 이벤트(조건부), 상점(고가 구매)
- 한 런 획득량: 약 6~10개
- 총 수량: 24종 (프로토타입 기준)
- 등급 출현비: Common 50% / Uncommon 35% / Rare 15%

### 8.2 Damage relics

| Name       | Rarity   | Effect                                                       |

## 9. Meta Progression (Slay the Spire structure)

### 9.1 Act structure
- 총 구성: **3액트 + 최종 보스**
- 액트당 예상 시간: 10~15분
- 액트당 노드 수: 12~15개

### 9.2 Node types

| Node       | Per act | Description                          | Reward                               |
|------------|---------|--------------------------------------|--------------------------------------|
| 일반 전투  | 7~8     | 벽돌 필드 클리어. 5~7턴               | 스펠 3개 중 택 1 + 골드              |
| 엘리트     | 1~2     | 특수 조건 + 강한 벽돌 배치. 8~12턴    | 유물 2개 중 택 1 + 레어 스펠         |
| 이벤트 (?) | 2       | 선택지 기반 리스크 / 리워드           | 조건부 유물 / 스펠 / HP             |
| 상점       | 1~2     | 스펠 구매, 슬롯 추가, 스펠 제거, HP 회복 | 골드 소비                          |
| 보스       | 1       | 고유 패턴의 보스 벽돌. 12~15턴        | 유물 3개 중 택 1 + 골드              |
| 휴식       | 1       | HP 회복 또는 스펠 업그레이드 택 1     | 선택                                  |

### 9.3 Combat duration reference

| Combat type | Turns   | Time  | Brick characteristics                |
|-------------|---------|-------|--------------------------------------|
| 일반 전투   | 5~7     | 1~2분 | 빈 칸 많음, HP 1 위주                 |
| 엘리트      | 8~12    | 2~3분 | 특수 벽돌 높은 비율, 고HP             |
| 보스        | 12~15   | 3~4분 | 보스 벽돌 + 고유 생성 패턴            |

### 9.4 Balance baseline
- 한 턴 플레이어 처리 가능 벽돌: 평균 3~5개 (스펠 빌드에 따라 편차)
- 매 턴 생성되는 벽돌: 일반 전투 기준 3~5개 / 줄 (8칸 중 빈 칸 포함)
- 처리량 ≈ 생성량이면 현상 유지. 처리량 > 생성량이면 클리어에 근접.
- 좋은 빌드일수록 빠르게 클리어, 약한 빌드는 턴이 길어지며 압박 누적.

## 10. Build Examples

유물과 스펠 조합으로 형성되는 빌드 방향성 예시입니다.

### 10.1 Pinball Machine
- **Spells**: 스파크 + 가속 + 바운스 마스터 + 리코셰
- **Relics**: 탄성 코팅 + 벽면 강화 + 무한 반사 + 탄약 공장
- **Core fantasy**: 바운스를 최대한 많이 해서 데미지를 쌓고, 소멸 시 추가 발사체 생성. "오래 살아남는 공"이 핵심.

### 10.2 One Shot One Kill
- **Spells**: 캐논볼 + 집중 + 강화탄 + 파쇄
- **Relics**: 처형자 + 근접 사격 + 궤적 잔상 + 위기 집중
- **Core fantasy**: 발사체 1개에 극단적 데미지 집중. 정밀 조준이 모든 것을 결정.

### 10.3 Bullet Hell
- **Spells**: 더블 샷 + 분열 + 충격파 + 속사
- **Relics**: 탄막 + 파편화 + 잔탄 소진 + 추가 탄약
- **Core fantasy**: 발사체 수로 밀어붙이는 구조. 개별 데미지는 낮지만 화면을 발사체로 채움.
